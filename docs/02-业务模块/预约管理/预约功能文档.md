# 泳课预约系统 - 预约功能文档

## 1. 预约创建流程

### 1.1 预约入口
- **学员详情页**: 点击"📅 预约"按钮进入预约页面
- **路由**: `/student/{studentId}/add-appointment`

### 1.2 预约表单字段

#### 1.2.1 日期选择
- **必填字段**
- **选择范围**: 今天至未来30天
- **交互方式**: 日历选择器

#### 1.2.2 时间选择
- **必填字段**
- **可用时间段**: 09:00, 10:00, 11:00, 12:00, 13:00, 14:00, 15:00, 16:00, 17:00
- **交互方式**: 点击选择，选中后立即高亮显示

#### 1.2.3 课程选择
- **自动过滤**: 根据选中的日期和时间筛选可用课程
- **显示信息**:
  - 课程名称
  - 课程开始时间
  - 支持点击"查看详情"查看完整信息

#### 1.2.4 学员信息
- **自动填充**: 学员姓名
- **显示信息**: 剩余课时数

#### 1.2.5 备注
- **选填字段**
- **字符限制**: 最多输入50字

### 1.3 表单验证规则
- 日期必选
- 时间必选
- 课程必选
- 剩余课时必须大于0

### 1.4 提交数据结构
```json
{
  "student_id": "学生ID",
  "course_id": "课程ID",
  "appointment_date": "2025-12-08",
  "appointment_time": "09:00",
  "notes": "备注内容",
  "lesson_consumed": false
}
```

## 2. 预约显示

### 2.1 学员详情页 - 上课记录
- **位置**: 学员详情页底部"📚 上课记录"区域
- **显示字段**:
  - **日期**: MM/DD 格式 (如: 12/08)
  - **时间**: HH:mm 格式 (如: 09:00)
  - **状态**: "已预约" / "已完成"
  - **课程消耗**:
    - 未消耗: "- → 0"
    - 已消耗: "- → 1"

### 2.2 排序规则
- **排序依据**: 按预约创建时间倒序排列
- **显示顺序**: 最新的预约记录显示在最上方

## 3. 预约状态管理

### 3.1 状态类型

#### 3.1.1 scheduled (已预约)
- **前端显示**: "已预约"
- **课程消耗显示**: "- → 0"
- **业务含义**: 预约已创建，但课程尚未进行

#### 3.1.2 completed (已完成)
- **前端显示**: "已完成"
- **课程消耗显示**: "- → 1"
- **业务含义**: 课程已完成，课时已消耗

### 3.2 状态转换流程
```
创建预约 → scheduled (已预约) → 课程进行 → completed (已完成)
```

### 3.3 课时消耗逻辑
- **预约创建时**: 默认 `lesson_consumed: false`
- **课程完成时**: 设置 `lesson_consumed: true`
- **自动扣费**: 完成课程后自动扣除相应套餐课时

## 4. 时间处理

### 4.1 时区处理
- **前端显示**: 使用本地时区
- **数据存储**: ISO格式字符串，无时区转换
- **时间准确性**: 确保用户选择的时间准确显示，避免时区偏移

### 4.2 日期格式规范
- **显示格式**:
  - 日期: MM/DD (12/08)
  - 时间: HH:mm (09:00)
  - 完整时间: yyyy-MM-dd HH:mm:ss
- **存储格式**: ISO字符串 (2025-12-08T09:00:00)

### 4.3 时间选择限制
- **最早时间**: 当天
- **最晚时间**: 未来30天
- **可选时段**: 09:00-17:00，每小时一个时段

## 5. 用户交互体验

### 5.1 交互流程
```
学员详情页 → 点击预约按钮 → 选择日期 → 选择时间 →
选择课程 → 查看学员信息 → 填写备注 → 提交预约
```

### 5.2 响应式设计
- **移动优先**: 固定最大宽度430px
- **触摸优化**: 按钮和输入框适合手指操作
- **视觉反馈**: 选择状态即时显示

### 5.3 错误处理
- **实时验证**: 表单字段输入时实时验证
- **友好提示**: 错误信息清晰易懂
- **网络异常**: 显示网络错误提示，支持重试

### 5.4 导航逻辑
- **智能返回**:
  - 优先返回上一个页面
  - 其次使用浏览器后退功能
- **状态保持**: 切换页面后返回时保持用户输入状态

## 6. 业务逻辑

### 6.1 预约冲突检测
- **时间冲突**: 同一学员同一时间段不能重复预约
- **课程容量**: 检查课程是否还有可用名额
- **课时充足**: 检查学员剩余课时是否足够

### 6.2 预约确认流程
1. **前端验证**: 基本信息完整性检查
2. **后端验证**: 业务规则验证
3. **数据保存**: 创建预约记录
4. **状态更新**: 更新学员课时状态
5. **通知机制**: 可选的预约成功通知

### 6.3 预约取消逻辑
- **取消条件**: 只能取消未来的预约
- **课时恢复**: 取消后恢复相应课时
- **状态更新**: 更新预约状态为已取消

## 7. 数据模型

### 7.1 预约记录结构
```json
{
  "_id": "预约ID",
  "student_id": "学生ID",
  "course_id": "课程ID",
  "appointment_date": "预约日期",
  "appointment_time": "预约时间",
  "notes": "备注",
  "lesson_consumed": false,
  "status": "scheduled",
  "create_time": "创建时间",
  "update_time": "更新时间"
}
```

### 7.2 关联数据
- **学员信息**: 姓名、剩余课时、套餐信息
- **课程信息**: 课程名称、时间、地点、教练信息

## 8. 技术实现

### 8.1 前端技术
- **Vue 3** + **Composition API**
- **响应式数据**: 实时表单验证和状态管理
- **路由管理**: Vue Router 处理页面导航

### 8.2 后端API
- **FastAPI**: RESTful API接口
- **数据验证**: Pydantic 模型验证
- **业务逻辑**: 预约冲突检测和课时管理

### 8.3 数据存储
- **MongoDB**: 文档数据库
- **索引优化**: 按学生ID和时间建立索引
- **事务处理**: 确保数据一致性
