# 泳课预约系统 - 套餐功能文档

## 1. 套餐类型

### 1.1 记次套餐
- **计费方式**: 按次计费
- **适用场景**: 固定课程数量的教学
- **课程类型**: 1v1, 1v2, 1v3, 1v5等

#### 核心字段
- **课程总数** (total_lessons): 套餐包含的总课程数
- **剩余课时** (remaining_lessons): 当前剩余可用课程数
- **套餐价格**: 用户购买价格
- **上交俱乐部金额**: 需要分给场地的费用

### 1.2 时长套餐
- **计费方式**: 按时间计费
- **适用场景**: 限定时间内的无限次课程
- **课程类型**: 统一显示为"时长套餐"

#### 核心字段
- **开始日期** (start_date): 套餐生效日期
- **结束日期** (end_date): 套餐失效日期
- **套餐价格**: 用户购买价格
- **上交俱乐部金额**: 需要分给场地的费用

## 2. 套餐创建流程

### 2.1 创建入口
- **学员详情页**: 点击"+ 新增套餐"按钮
- **路由**: `/student/{studentId}/add-package`

### 2.2 套餐表单

#### 2.2.1 套餐类别选择
- **选择方式**: 单选按钮
- **选项**: 记次套餐 / 时长套餐
- **动态切换**: 选择后表单字段动态变化

#### 2.2.2 记次套餐配置

**课程类型设置**
- **输入方式**: 文本输入框
- **快速选择**: 1v1, 1v2, 1v3, 1v5 等预设选项
- **自动生成**: 套餐名称格式 "{课程类型} 记次套餐"

**课程总数设置**
- **输入方式**: 数字输入框
- **快速选择**: 12节等预设选项
- **验证规则**: 必须大于0

#### 2.2.3 时长套餐配置

**日期设置**
- **开始日期**: 日期选择器，默认为当天
- **结束日期**:
  - 月卡按钮: 点击自动计算结束日期为开始日期+1个月
  - 手动选择: 支持自定义结束日期

**套餐名称**
- **自动生成**: 统一显示为"时长套餐"

#### 2.2.4 价格信息设置

**售价配置**
- **输入方式**: 数字输入框
- **精度**: 支持小数点后2位
- **验证**: 必须大于0

**上交俱乐部金额**
- **输入方式**: 数字输入框
- **快速选择**: 600元等预设金额
- **验证**: 必须大于等于0

**利润计算**
- **自动计算**: 售价 - 上交俱乐部金额
- **实时显示**: 输入价格后立即显示

### 2.3 表单验证规则
- 套餐类别必选
- 价格信息必填
- 记次套餐需设置课程总数
- 时长套餐需设置结束日期

## 4. 套餐显示

### 4.1 学员详情页 - 套餐信息
- **位置**: 学员详情页"📦 套餐信息"区域
- **显示字段**:
  - **套餐名称**: 自动生成的套餐名称
  - **剩余信息**:
    - 记次套餐: "剩余X次"
    - 时长套餐: 显示有效期信息
  - **套餐类型**: 一对一/一对二/一对三
  - **计费方式**: 记次套餐/时长套餐
  - **课程余量**: "X/X节" (记次套餐)
  - **套餐价格**: "¥XXXX"

### 4.2 套餐编辑页
- **路由**: `/packages/{packageId}/edit`
- **显示模式**: 只读模式，所有字段不可编辑
- **显示内容**:
  - **套餐类型选择器**: 禁用状态，显示当前套餐类型
  - **价格信息**: 售价、上交俱乐部金额、计算利润
  - **创建时间**: 横向显示，格式化时间字符串

## 5. 套餐状态管理

### 5.1 初始状态
- **记次套餐**:
  - remaining_lessons = total_lessons
  - 显示状态为全新套餐
- **时长套餐**:
  - start_date = 创建日期
  - end_date = 根据套餐类型计算

### 5.2 课时消耗逻辑
- **触发条件**: 预约完成且 lesson_consumed = true
- **扣费规则**:
  - 自动选择剩余课时最多的记次套餐
  - 扣减该套餐的 remaining_lessons
- **记录更新**: 更新套餐的 update_time

### 5.3 套餐状态检查
- **有效性检查**:
  - 记次套餐: remaining_lessons > 0
  - 时长套餐: 当前日期在 start_date 和 end_date 之间
- **显示状态**:
  - 有效: 正常显示
  - 无效: 标记为已过期或已用完

## 6. 套餐逻辑特性

### 6.1 类型切换状态保存
- **问题场景**: 在记次套餐和时长套餐之间切换时，课程类型信息丢失
- **解决方案**:
  - 切换前保存当前套餐类型的课程设置
  - 切换回原类型时恢复保存的设置
  - 记次套餐和时长套餐的课程类型互不影响

### 6.2 智能套餐选择
- **选择策略**: 创建预约时自动选择合适的套餐
- **优先级**:
  1. 剩余课时最多的记次套餐
  2. 在有效期内的时长套餐
- **用户可见**: 显示所选套餐的信息

### 6.3 套餐组合
- **支持场景**: 同一学员可以拥有多个不同类型的套餐
- **显示规则**: 按创建时间倒序显示所有有效套餐
- **扣费顺序**: 按剩余课时从多到少依次扣费

## 7. 套餐管理

- **按学员查询**: 获取指定学员的所有套餐
- **按状态过滤**: 筛选有效/无效套餐
- **排序规则**: 按创建时间倒序排列

### 7.3 套餐统计
- **课时统计**: 计算学员总课时和已用课时
- **套餐价值**: 计算已购买套餐总价值
- **使用率**: 计算课时使用率

## 9. 业务规则

### 9.1 套餐定价
- **价格验证**: 售价必须大于等于上交俱乐部金额
- **利润计算**: 自动计算并显示利润
- **价格精度**: 支持小数点后2位

### 9.2 套餐有效期
- **记次套餐**: 没有时间限制，用完为止
- **时长套餐**: 严格按有效期控制使用

### 9.3 套餐转让
- **当前状态**: 暂不支持套餐转让功能
- **未来规划**: 可考虑添加套餐转让功能
