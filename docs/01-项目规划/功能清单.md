# Easy Book - 实际功能清单

**更新时间**: 2025-12-03
**版本**: v1.1.0

## 📋 概述

本文档描述Easy Book泳课学员管理系统的**实际实现功能**，作为所有开发、测试、部署的权威参考。

## 🚀 系统架构

### 后端 (FastAPI + MongoDB)
- **运行端口**: 8002 (重要：不是8000)
- **数据库**: MongoDB
- **API文档**: http://localhost:8002/docs
- **健康检查**: http://localhost:8002/health

### 前端 (Vue 3 + Vite)
- **运行端口**: 5173 (开发环境)
- **UI框架**: 自定义移动端UI (未使用Element Plus)
- **状态管理**: Pinia
- **路由**: Vue Router 4

## 🛠️ 实际API端点

### 基础端点
- `GET /` - 根路径
- `GET /health` - 系统健康检查
- `GET /api/health/db` - 数据库连接检查

### 学生管理 (`/api/students/`)
- `POST /` - 创建学生 ✅
- `GET /` - 获取学生列表 ✅
- `GET /{student_id}` - 获取单个学生 ✅
- `PUT /{student_id}` - 更新学生信息 ✅
- `DELETE /{student_id}` - 删除学生 ✅

### 预约管理 (`/api/appointments/`)
- `POST /` - 创建预约 ✅
- `GET /student/{student_id}` - 获取学生预约记录 ✅
- `GET /daily/{target_date}` - 获取指定日期预约 ✅
- `GET /upcoming` - 获取未来预约列表 ✅
- `PUT /{appointment_id}` - 更新预约 ✅
- `DELETE /{appointment_id}` - 删除预约 ✅

### 考勤管理 (`/api/attendance/`)
- `POST /checkin` - 学生签到 ✅
- `POST /absent` - 标记缺席 ✅
- `GET /student/{student_id}` - 获取学生考勤记录 ✅

## 🎯 业务规则

### 预约冲突规则
- ✅ **1v1课程**: 同一时间段不能有多个1v1预约
- ✅ **1v多课程**: 可以与1v1课程在同一时间 (取决于具体实现)
- ✅ **同一学生**: 不能在同一时间有多个预约

### 课程扣费规则
- ✅ **签到**: 剩余课程数 -1
- ✅ **缺席**: 剩余课程数不变
- ✅ **课程计算**: `attended_lessons = total_lessons - remaining_lessons`
