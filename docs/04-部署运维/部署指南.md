# Easy Book 部署指南

## 部署架构

本应用采用前后端分离架构：
- **前端**：Vue.js + Vite（静态文件）
- **后端**：FastAPI + Python（API服务）
- **数据库**：MongoDB

## 部署方式

### 方式一：传统服务器部署

#### 1. 服务器环境准备

```bash
# 安装 Python 3.11+
sudo apt update
sudo apt install python3.11 python3.11-venv python3.11-pip

# 安装 Node.js 18+
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 PM2（进程管理器）
sudo npm install -g pm2
```

#### 2. 数据库配置

**使用 MongoDB Atlas（推荐）**

1. 访问 https://www.mongodb.com/atlas 创建免费集群
2. 创建数据库用户
3. 获取连接字符串，格式类似：
   ```
   mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/easy_book?retryWrites=true&w=majority
   ```

**使用自建 MongoDB**

```bash
# 安装 MongoDB
wget -qO - https://www.mongodb.org/static/pgp/server-7.0.asc | sudo apt-key add -
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list
sudo apt-get update
sudo apt-get install -y mongodb-org

# 启动 MongoDB
sudo systemctl start mongod
sudo systemctl enable mongod
```

#### 3. 后端部署

```bash
# 上传后端代码到服务器
scp -r backend/ user@your-server:/path/to/app/

# 登录服务器
ssh user@your-server

# 进入后端目录
cd /path/to/app/backend

# 创建虚拟环境
python3.11 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 创建环境变量文件
cat > .env << EOF
MONGODB_URL=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/easy_book?retryWrites=true&w=majority
DB_NAME=easy_book
EOF

# 测试后端
python run.py

# 使用 PM2 管理进程
pm2 start run.py --name "easy-book-backend" --interpreter python
pm2 save
pm2 startup
```

#### 4. 前端部署

```bash
# 上传前端代码
scp -r frontend/ user@your-server:/path/to/app/

# 登录服务器并构建前端
ssh user@your-server
cd /path/to/app/frontend

# 安装依赖
npm install

# 修改 API 地址（如果需要）
# 编辑 vite.config.js 中的 proxy 配置

# 构建生产版本
npm run build

# 使用 Nginx 或 Apache 托管静态文件
sudo cp -r dist/* /var/www/html/
```

#### 5. Nginx 配置

```nginx
# /etc/nginx/sites-available/easy-book
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://localhost:8002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```bash
# 启用站点
sudo ln -s /etc/nginx/sites-available/easy-book /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 方式二：Docker 部署

#### 1. 创建 Dockerfile

**后端 Dockerfile**
```dockerfile
# backend/Dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8002

CMD ["python", "run.py"]
```

**前端 Dockerfile**
```dockerfile
# frontend/Dockerfile
FROM node:18-alpine as build

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf

EXPOSE 80
```

#### 2. 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  mongodb:
    image: mongo:6
    restart: always
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: password
      MONGO_INITDB_DATABASE: easy_book
    volumes:
      - mongodb_data:/data/db
    ports:
      - "27017:27017"

  backend:
    build: ./backend
    restart: always
    environment:
      MONGODB_URL: mongodb://admin:password@mongodb:27017/easy_book?authSource=admin
      DB_NAME: easy_book
    depends_on:
      - mongodb
    ports:
      - "8002:8002"

  frontend:
    build: ./frontend
    restart: always
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mongodb_data:
```

#### 3. 部署命令

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 方式三：云平台部署

#### 1. Vercel + Railway 部署

**前端部署到 Vercel**

```bash
# 安装 Vercel CLI
npm i -g vercel

# 在前端目录
cd frontend
vercel --prod
```

**后端部署到 Railway**

```bash
# 安装 Railway CLI
npm install -g @railway/cli

# 登录并部署
railway login
railway init
railway up
```

#### 2. 环境变量配置

在 Railway 控制台中设置环境变量：
```
MONGODB_URL=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/easy_book
DB_NAME=easy_book
PORT=8002
```

## 环境变量说明

创建 `.env` 文件在项目根目录：

```env
# 数据库配置
MONGODB_URL=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/easy_book?retryWrites=true&w=majority
DB_NAME=easy_book

# 可选：服务器配置
HOST=0.0.0.0
PORT=8002
```

## 安全建议

1. 使用强密码和安全的数据库连接
2. 启用防火墙，只开放必要端口
3. 定期更新系统和依赖
4. 使用 HTTPS
5. 设置访问频率限制
6. 定期备份数据
