## 📋 常见问题分类

### 🔴 严重问题（立即处理）

#### 问题1：完全无法访问网站
**症状**: 浏览器显示"无法连接"或"连接超时"

**检查步骤**:
```bash
# 1. 检查服务器是否在线
ping 49.233.60.29

# 2. 检查SSH连接
ssh -i C:/mysofts/keys/qcloud_lighthouse_beijing ubuntu@49.233.60.29

# 3. 检查防火墙
sudo ufw status
sudo iptables -L

# 4. 检查nginx状态
sudo systemctl status nginx
sudo journalctl -u nginx --no-pager -n 20
```

**解决方案**:
- 防火墙问题：`sudo ufw allow 80/tcp`
- nginx未启动：`sudo systemctl start nginx`
- 服务器宕机：重启服务器或联系服务商

#### 问题2：后端API完全不响应
**症状**: API请求返回502或504错误

**检查步骤**:
```bash
# 1. 检查后端进程
ps aux | grep "python3 run.py" | grep -v grep

# 2. 检查后端端口
sudo netstat -tlnp | grep :8002

# 3. 检查后端日志
tail -50 /tmp/backend.log

# 4. 直接测试后端
curl -s http://localhost:8002/health
```

**解决方案**:
```bash
# 重启后端服务
pkill -f "python3 run.py"
cd /home/ubuntu/backend
nohup python3 run.py 8002 > /tmp/backend.log 2>&1 &

# 检查端口冲突
sudo lsof -i :8002
kill -9 <占用端口的进程ID>
```

### 🟡 中等问题（需要处理）

#### 问题3：前端页面显示旧版本
**症状**: 页面功能不更新，仍然显示旧版本

**检查步骤**:
```bash
# 1. 检查nginx配置指向
sudo grep -n "root /var/www/html" /etc/nginx/sites-enabled/easy-book.conf

# 2. 检查文件权限
ls -la /var/www/html/
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html

# 3. 强制刷新nginx缓存
sudo systemctl reload nginx
sudo nginx -s reload
```

**解决方案**:
- 清除浏览器缓存（Ctrl+F5）
- 确认文件部署到正确目录
- 重新加载nginx配置

#### 问题4：数据库连接失败
**症状**: API返回数据库连接错误

**检查步骤**:
```bash
# 1. 测试数据库连接
mongo mongodb://easy-book:REDACTED_APP_PW@10.0.20.14:27017/easy_book?authSource=admin

# 2. 检查环境变量
cat /home/ubuntu/backend/.env

# 3. 检查数据库日志
tail -20 /tmp/backend.log | grep -i "mongo\|database"
```

**解决方案**:
```bash
# 更新环境变量
cat > /home/ubuntu/backend/.env << 'EOF'
MONGODB_URL=mongodb://easy-book:REDACTED_APP_PW@10.0.20.14:27017/easy_book?authSource=admin
DB_NAME=easy_book
EOF

# 重启后端服务
pkill -f "python3 run.py"
cd /home/ubuntu/backend && nohup python3 run.py 8002 > /tmp/backend.log 2>&1 &
```

### 🟢 轻微问题（可以稍后处理）

#### 问题5：部分功能异常
**症状**: 特定功能不工作，其他功能正常

**检查步骤**:
```bash
# 1. 检查API响应
curl -s http://49.233.60.29/api/students/ | jq .
curl -s http://49.233.60.29/api/appointments/upcoming?days=7 | jq .

# 2. 检查前端控制台错误
# 在浏览器开发者工具中查看Console和Network

# 3. 检查特定功能日志
tail -f /tmp/backend.log | grep -i "error\|exception"
```

**常见功能问题**:
- **签到功能不工作**: 检查考勤API和状态更新逻辑
- **预约功能异常**: 检查冲突检测和数据库写入
- **日历显示问题**: 检查时间格式化和状态计算

## 🔍 日志分析

### 后端日志位置
```bash
# 主日志文件
tail -f /tmp/backend.log

# 系统日志
sudo journalctl -u nginx --follow
sudo journalctl --follow | grep "python\|mongo"
```

### 常见错误模式

#### 数据库连接错误
```
pymongo.errors.ConnectionFailure: ...
MongoDB connection failed
```
**原因**: 数据库服务不可达或认证失败

#### 端口占用错误
```
Address already in use: 8002
```
**原因**: 端口被其他进程占用

#### 权限错误
```
Permission denied: '/var/www/html'
```
**原因**: 文件权限配置不正确

#### 内存不足
```
MemoryError: ...
Out of memory
```
**原因**: 系统内存不足

## 🔄 恢复和回滚

### 完整系统恢复
```bash
# 1. 从备份恢复数据库
mongorestore "mongodb://easy-book:REDACTED_APP_PW@10.0.20.14:27017/easy_book?authSource=admin" /backup/20241201/db

# 2. 从备份恢复代码
cp -r /backup/backend_20241201 /home/ubuntu/backend
cp -r /backup/html_20241201/* /var/www/html/

# 3. 重启所有服务
sudo systemctl restart nginx
pkill -f "python3 run.py"
cd /home/ubuntu/backend && nohup python3 run.py 8002 > /tmp/backend.log 2>&1 &
```

### 快速回滚到上一版本
```bash
# 回滚到最近的备份
cd /home/ubuntu
cp -r backend_backup_*/ backend
cp -r html_backup_*/* /var/www/html/

# 重启服务
sudo systemctl reload nginx
pkill -f "python3 run.py"
cd backend && nohup python3 run.py 8002 > /tmp/backend.log 2>&1 &
```
