# 部署准备清单

> 🎯 **首次部署必读** - 在开始部署前，请逐项检查完成

## 🔧 环境要求

### 服务器要求
- [ ] **操作系统**: Ubuntu 18.04+ / CentOS 7+ / Debian 9+
- [ ] **内存**: 最低 2GB，推荐 4GB
- [ ] **存储**: 最低 20GB 可用空间
- [ ] **网络**: 稳定的互联网连接
- [ ] **权限**: sudo 权限（用于安装软件和配置服务）

### 本地开发机要求
- [ ] **Python**: 3.11+
- [ ] **Node.js**: 18+
- [ ] **Git**: 用于代码版本管理
- [ ] **SSH客户端**: 用于连接服务器

## 📋 部署前准备工作

### 1. 服务器准备
- [ ] **购买服务器**（如：腾讯云轻量服务器）
- [ ] **获取服务器IP**: `49.233.60.29`（示例）
- [ ] **配置安全组**:
  - 开放端口：80 (HTTP), 443 (HTTPS), 22 (SSH), 8002 (后端API)
- [ ] **设置SSH密钥**：`C:/mysofts/keys/qcloud_lighthouse_beijing`

### 2. 数据库准备
- [ ] **MongoDB实例**：
  - 本地测试：`localhost:27017`
  - 生产环境：`10.0.20.14:27017`
- [ ] **创建数据库用户**：
  ```bash
  # 用户名: easy-book
  # 密码: REDACTED_APP_PW
  # 数据库: easy_book
  ```

### 3. 域名准备（可选）
- [ ] **域名解析**：将域名指向服务器IP
- [ ] **SSL证书**：配置HTTPS（可选但推荐）

### 4. 代码准备
- [ ] **克隆项目**：
  ```bash
  git clone <repository-url>
  cd easy-book
  ```
- [ ] **本地测试**：
  ```bash
  # 启动后端测试
  cd backend && python -m api_server.main 8002
  curl -s http://localhost:8002/health

  # 启动前端测试
  cd frontend && npm run dev
  ```

├── backend/           # 后端代码

## ⚠️ 安全检查清单

### 服务器安全
- [ ] **更改默认SSH端口**（可选，但推荐）
- [ ] **配置防火墙**：
  ```bash
  sudo ufw enable
  sudo ufw allow 22/tcp
  sudo ufw allow 80/tcp
  sudo ufw allow 443/tcp
  sudo ufw allow 8002/tcp
  ```
- [ ] **禁用root登录**（推荐）
- [ ] **配置fail2ban**（推荐）

### 数据库安全
- [ ] **使用强密码**
- [ ] **限制数据库访问IP**
- [ ] **启用数据库认证**
- [ ] **定期备份数据**

### 应用安全
- [ ] **检查敏感信息**：确保不在代码中包含密码
- [ ] **配置环境变量**：所有敏感配置通过.env文件
- [ ] **HTTPS配置**：生产环境启用SSL
