## 🚀 更新流程

### 第1步：准备更新包

#### 本地准备
```bash
# 1. 确认代码最新
git pull origin master
git status

# 2. 运行本地测试
cd backend && python -m api_server.main 8002 &
curl -s http://localhost:8002/health

# 3. 打包后端代码
tar -czf backend-update-v{版本号}.tar.gz -C backend api_server

# 4. 打包前端代码
cd frontend && npm run build
tar -czf frontend-update-v{版本号}.tar.gz dist/
```

### 第2步：上传更新包
```bash
# 上传到服务器
scp -i C:/mysofts/keys/qcloud_lighthouse_beijing backend-update-v{版本号}.tar.gz ubuntu@49.233.60.29:/home/ubuntu/
scp -i C:/mysofts/keys/qcloud_lighthouse_beijing frontend-update-v{版本号}.tar.gz ubuntu@49.233.60.29:/home/ubuntu/
```

### 第3步：服务器端更新

#### 连接服务器
```bash
ssh -i C:/mysofts/keys/qcloud_lighthouse_beijing ubuntu@49.233.60.29
cd /home/ubuntu
```

#### 后端更新
```bash
# 1. 解压新后端代码
tar -xzf backend-update-v{版本号}.tar.gz

# 2. 停止当前后端服务
pkill -f "python3 run.py"

# 3. 备份并替换后端代码
cp -r backend backend_backup_$(date +%Y%m%d_%H%M%S)
cp -r api_server backend/

# 4. 更新环境配置（如需要）
cat > backend/.env << 'EOF'
MONGODB_URL=mongodb://easy-book:REDACTED_APP_PW@10.0.20.14:27017/easy_book?authSource=admin
DB_NAME=easy_book
EOF

# 5. 安装新的Python依赖（如有变化）
cd backend
pip3 install -r requirements.txt

# 6. 启动后端服务
nohup python3 run.py 8002 > /tmp/backend.log 2>&1 &

# 7. 验证后端服务
sleep 5
ps aux | grep "python3 run.py" | grep -v grep
curl -s http://localhost:8002/health
```

#### 前端更新
```bash
# 1. 解压新前端代码
tar -xzf frontend-update-v{版本号}.tar.gz

# 2. 备份并部署新前端
sudo cp -r /var/www/html /var/www/html_backup_$(date +%Y%m%d_%H%M%S)
sudo cp -r dist/* /var/www/html/

# 3. 设置正确的文件权限
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html

# 4. 重新加载nginx
sudo systemctl reload nginx

# 5. 验证前端服务
curl -s http://localhost/ | head -5
```

## 📊 版本管理

### 版本记录
每次更新后记录版本信息：
```bash
# 在/home/ubuntu/version.log中记录
echo "v{版本号} - {更新日期} - {更新内容}" >> /home/ubuntu/version.log

# 查看版本历史
cat /home/ubuntu/version.log
```

### 版本号规则
- **格式**: v{主版本}.{次版本}.{修订版本}
- **主版本**: 重大架构变更
- **次版本**: 新功能添加
- **修订版本**: Bug修复和小改进

## 🔧 自动化脚本（可选）

### 更新脚本示例
创建 `/home/ubuntu/update.sh`:
```bash
#!/bin/bash
set -e

VERSION=$1
if [ -z "$VERSION" ]; then
    echo "用法: ./update.sh v1.2.0"
    exit 1
fi

echo "开始更新到版本 $VERSION"

# 备份当前版本
echo "备份当前版本..."
cp -r backend backend_backup_$(date +%Y%m%d_%H%M%S)
cp -r /var/www/html html_backup_$(date +%Y%m%d_%H%M%S)

# 更新后端
echo "更新后端..."
tar -xzf backend-update-$VERSION.tar.gz
pkill -f "python3 run.py" || true
cp -r api_server backend/
cd backend && nohup python3 run.py 8002 > /tmp/backend.log 2>&1 &

# 更新前端
echo "更新前端..."
tar -xzf frontend-update-$VERSION.tar.gz
sudo cp -r dist/* /var/www/html/
sudo chown -R www-data:www-data /var/www/html
sudo systemctl reload nginx

# 验证
echo "验证更新..."
sleep 5
if curl -s http://localhost:8002/health > /dev/null; then
    echo "✅ 后端更新成功"
else
    echo "❌ 后端更新失败，请手动检查"
    exit 1
fi

echo "✅ 更新完成"
```
