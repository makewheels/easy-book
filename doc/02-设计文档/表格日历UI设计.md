# 表格日历功能UI设计文档

## 一、设计理念

### 1. 设计原则
- **简洁性**：界面简洁明了，避免信息过载
- **一致性**：与现有系统保持统一的视觉风格
- **易用性**：操作直观，符合用户习惯
- **移动优先**：优先考虑移动端用户体验

### 2. 视觉风格
- **主色调**：继续使用现有的 #1989fa 蓝色主题
- **字体**：保持系统字体，确保可读性
- **圆角**：使用 8px 圆角保持现代感
- **阴影**：适度使用阴影增强层次感

## 二、页面布局设计

### 1. 整体布局结构

```
┌─────────────────────────────────────┐
│           顶部导航栏                │
├─────────────────────────────────────┤
│         日期控制区域                │
├─────────────────────────────────────┤
│                                     │
│         日历表格主体                │
│                                     │
├─────────────────────────────────────┤
│         底部导航栏                  │
└─────────────────────────────────────┘
```

### 2. 顶部导航栏
- **高度**：60px
- **背景**：#1989fa（与现有系统一致）
- **标题**："课程日历" 或 "排课表"
- **返回按钮**：左侧，使用标准返回图标
- **右侧操作**：可添加"今天"快捷按钮

### 3. 日期控制区域
- **高度**：50px
- **背景**：白色
- **边框**：底部 1px solid #eee
- **布局**：左右导航 + 中间日期显示

#### 日期导航按钮
- **左按钮**："< 上周"
- **右按钮**："下周 >"
- **按钮样式**：蓝色文字，无边框背景
- **点击区域**：最小 44px × 44px

#### 日期显示
- **居中显示**：当前显示的日期范围
- **格式**："MM月DD日 - MM月DD日" 或 "X月X日-X日"
- **字体大小**：16px，加粗显示

#### 快捷按钮组
- **本周**："本周"
- **下周**："下周"
- **今天**："今天"
- **样式**：灰色背景，蓝色文字，圆角按钮

### 4. 日历表格主体

#### 表格头部
- **高度**：40px
- **背景**：#f5f5f5
- **列宽**：时间列 80px，日期列均分剩余宽度

| 时间 | 周一(12/2) | 周二(12/3) | ... | 周日(12/8) |
|------|------------|------------|-----|------------|

#### 时间列
- **背景**：#f9f9f9
- **字体**：14px，居中对齐
- **行高**：60px
- **边框**：右侧 1px solid #eee

#### 日期列
- **表头**：显示星期和日期
- **今天是哪天**：用蓝色背景标识今日
- **周末**：可用淡色背景区分

#### 时间单元格
- **高度**：60px
- **内边距**：8px
- **边框**：1px solid #eee
- **背景**：白色
- **悬停**：浅灰色背景 #f9f9f9

### 5. 底部导航栏
- **高度**：60px
- **背景**：白色
- **位置**：固定底部
- **最大宽度**：430px（居中显示）
- **边框**：顶部 1px solid #eee

## 三、组件设计

### 1. 学生卡片组件

#### 单个学生显示
```vue
<div class="student-item">
  <span class="student-name">张三</span>
  <span class="package-type">1v1</span>
</div>
```

#### 样式规范
- **容器**：圆角 4px，内边距 4px 8px
- **学生姓名**：14px，黑色文字
- **套餐类型**：12px，彩色标签

#### 套餐类型颜色
- **1v1**：蓝色背景 #1989fa，白色文字
- **1v多**：绿色背景 #07c160，白色文字

#### 多个学生显示 - 垂直堆叠设计
```vue
<div class="multiple-students">
  <div class="student-card" :class="getStatusClass(student.status)" @click="goToStudent(student.id)">
    <div class="student-name">{{ student.name }}</div>
  </div>
  <div class="student-card status-checked" @click="goToStudent(student2.id)">
    <div class="student-name">{{ student2.name }}</div>
  </div>
</div>
```

#### 垂直堆叠设计规范
- **布局**：学生卡片垂直排列，每个占据一行
- **间距**：卡片之间2px间距
- **点击区域**：每个学生卡片独立可点击，最小44px高度
- **状态显示**：每个学生卡片显示独立的背景色状态
- **文字居中**：学生姓名在卡片中垂直水平居中

### 2. 状态标识组件

#### 学生状态颜色系统
基于高对比度设计，每个学生卡片显示独立的背景色状态：

- **未到上课时间 (scheduled)**：蓝色背景 #1890ff，黑色文字
- **已签到 (checked)**：绿色背景 #52c41a，黑色文字
- **已缺席 (absent)**：灰色背景 #8c8c8c，黑色文字

#### 状态显示规则
- **无状态文字**：不显示"已签到"、"已缺席"等文字标签
- **背景色区分**：通过背景色和文字颜色区分状态
- **字体加粗**：文字加粗（font-weight: 600）提高可读性
- **触摸优化**：最小44px点击区域，符合iOS设计规范

#### 冲突提醒
- **1v1冲突**：红色感叹号 ⚠️
- **位置**：单元格右上角
- **大小**：16px × 16px

### 3. 快捷操作组件

#### 点击单元格弹出菜单
```vue
<div class="cell-actions">
  <button @click="createAppointment">新增预约</button>
  <button @click="viewDetails">查看详情</button>
</div>
```

#### 长按手势
- **触发时间**：500ms
- **反馈**：震动（如果支持）
- **动作**：显示快捷菜单

## 四、响应式设计

### 1. 移动端优化（宽度 < 768px）

#### 表格适配
- **横向滚动**：支持左右滑动
- **固定时间列**：时间列保持固定，日期列可滚动
- **最小宽度**：时间列 80px，日期列 100px

```css
.calendar-container {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
}

.calendar-table {
  min-width: 780px; /* 确保表格不被压缩 */
}

.time-column {
  position: sticky;
  left: 0;
  z-index: 10;
  background: #f9f9f9;
}
```

#### 触摸优化
- **点击区域**：最小 44px × 44px
- **间距**：元素间最小间距 8px
- **手势支持**：左右滑动切换周

### 2. 平板端适配（768px - 1024px）

#### 表格布局
- **完整显示**：尝试完整显示所有列
- **字体大小**：可适当增大至 16px
- **行高**：可调整至 70px

### 3. 桌面端适配（> 1024px）

#### 居中布局
```css
.calendar-container {
  max-width: 1200px;
  margin: 0 auto;
}
```

## 五、交互动效

### 1. 页面切换动画
- **进入动画**：从右侧滑入，300ms ease-out
- **退出动画**：向左侧滑出，300ms ease-in

### 2. 日期切换动画
- **切换方向**：上周向左滑出，下周向右滑入
- **动画时长**：250ms ease-out
- **加载状态**：切换时显示加载指示器

### 3. 单元格交互
- **按下效果**：背景变暗，轻微缩放（0.98）
- **弹起效果**：恢复正常状态，100ms ease-out
- **点击反馈**：涟漪效果（如果支持）

### 4. 菜单动画
- **显示动画**：从点击位置展开，200ms ease-out
- **隐藏动画**：缩小消失，150ms ease-in

## 九、可访问性设计

### 1. 色彩对比度
- **正常文字**：对比度 ≥ 4.5:1
- **大文字**：对比度 ≥ 3:1
- **图标**：对比度 ≥ 3:1

### 2. 键盘导航
- **Tab 顺序**：逻辑清晰的焦点顺序
- **焦点样式**：明显的焦点指示器
- **快捷键**：支持方向键导航

### 3. 屏幕阅读器
- **语义化标签**：使用合适的 HTML 标签
- **ARIA 属性**：提供必要的辅助信息
- **替代文字**：图标提供文字说明

## 十、性能优化

### 1. 渲染优化
- **虚拟滚动**：大数据量时使用虚拟滚动
- **懒加载**：按需加载预约数据
- **缓存策略**：合理缓存已渲染的内容

### 2. 动画优化
- **GPU 加速**：使用 transform 和 opacity
- **避免重排**：减少布局变化
- **动画时长**：控制在 300ms 以内

### 3. 图片优化
- **图标使用 SVG**：矢量图标，无损缩放
- **按需加载**：非首屏图片延迟加载
- **压缩优化**：图片文件大小优化
