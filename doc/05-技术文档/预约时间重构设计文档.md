# 预约时间重构设计文档

## 1. 背景

当前系统使用 `appointment_date`（日期）和 `time_slot`（时间段）的组合来记录预约时间，所有课程都按1小时计算。这种设计存在以下问题：

1. **时间粒度固定**：所有课程都强制按1小时计算，无法支持非标准时长课程
2. **时间显示不够精确**：只能显示时间段起点，无法显示课程具体时长
3. **未来扩展性差**：未来可能需要支持1.5小时、2小时等不同时长的课程
4. **数据冗余**：时间段信息重复存储在多个相关表中

## 2. 设计目标

### 2.1 核心改进
- 将 `appointment_date` + `time_slot` 改为 `start_time` + `end_time`
- 支持任意时长的课程预约
- 提供更精确的时间显示和冲突检测
- 保持向后兼容性，确保现有数据可以平滑迁移

### 2.2 功能要求
- 支持1小时、1.5小时、2小时等常见课程时长
- 准确显示课程的开始和结束时间
- 改进时间冲突检测逻辑
- 优化日历视图的时间显示
- 保持数据一致性和完整性

## 3. 数据结构设计

### 3.1 新的数据模型

#### Appointment 表
```python
class AppointmentModel(BaseModel):

    update_time: datetime = Field(default_factory=datetime.utcnow)

    # 计算属性（不存储在数据库）
    duration_minutes: int = Field(default=0, description="课程时长（分钟）")
    duration_hours: float = Field(default=0.0, description="课程时长（小时）")
```

#### Attendance 表
```python
class AttendanceModel(BaseModel):
```

### 3.2 字段说明

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| start_time | datetime | 课程开始时间（精确到分钟） | 2025-12-04 14:00:00 |
| end_time | datetime | 课程结束时间（精确到分钟） | 2025-12-04 15:30:00 |
| duration_minutes | int | 课程时长（分钟，计算得出） | 90 |
| duration_hours | float | 课程时长（小时，计算得出） | 1.5 |

### 3.3 计算字段逻辑

```python
# 在模型中自动计算
@property
def duration_minutes(self) -> int:
    """计算课程时长（分钟）"""
    return int((self.end_time - self.start_time).total_seconds() / 60)

@property
def duration_hours(self) -> float:
    """计算课程时长（小时）"""
    return round(self.duration_minutes / 60, 2)
```

## 5. 数据迁移方案

### 5.1 迁移策略

1. **新增字段**：在 `appointments` 和 `attendances` 表中添加 `start_time` 和 `end_time` 字段
2. **数据转换**：将现有的 `appointment_date` + `time_slot` 组合转换为 `start_time` 和 `end_time`
3. **渐进迁移**：保留旧字段一段时间，逐步切换到新字段
4. **验证数据**：确保转换后的数据完整性和一致性

### 5.2 迁移脚本逻辑

```python
def migrate_appointment_data():
    """迁移预约数据到新的时间结构"""
    appointments = db.get_appointments()

    for apt in appointments:
        # 解析现有时间
        appointment_date = apt.get('appointment_date')  # "2025-12-04"
        time_slot = apt.get('time_slot')               # "14:00"

        # 转换为datetime对象
        start_time_str = f"{appointment_date} {time_slot}:00"
        start_time = datetime.fromisoformat(start_time_str)
        end_time = start_time + timedelta(hours=1)  # 默认1小时

        # 更新数据库
        db.update_appointment(apt['_id'], {
            'start_time': start_time,
            'end_time': end_time
        })
```

## 6. 前端改造方案

### 6.1 时间选择器组件

**新的时间选择器设计：**

```vue
<template>
  <div class="time-selector">
    <div class="form-group">
      <label>选择日期</label>
      <input type="date" v-model="selectedDate" />
    </div>

    <div class="form-group">
      <label>开始时间</label>
      <select v-model="startTime">
        <option value="07:00">07:00</option>
        <option value="08:00">08:00</option>
        <option value="09:00">09:00</option>
        <!-- ... -->
        <option value="19:00">19:00</option>
      </select>
    </div>

    <div class="form-group">
      <label>课程时长</label>
      <select v-model="duration" @change="calculateEndTime">
        <option value="60">1小时</option>
        <option value="90">1.5小时</option>
        <option value="120">2小时</option>
      </select>
    </div>

    <div class="form-group">
      <label>结束时间</label>
      <input type="text" :value="endTime" readonly />
    </div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      selectedDate: '',
      startTime: '',
      duration: 60,
      endTime: ''
    }
  },
  watch: {
    selectedDate() {
      this.updateEndTime()
    },
    startTime() {
      this.updateEndTime()
    }
  },
  methods: {
    calculateEndTime() {
      if (this.selectedDate && this.startTime) {
        const start = new Date(`${this.selectedDate}T${this.startTime}:00`)
        const end = new Date(start.getTime() + this.duration * 60 * 1000)
        this.endTime = this.formatTime(end)
      }
    },
    formatTime(date) {
      return date.toTimeString().slice(0, 5)
    }
  }
}
</script>
```

## 7. 业务逻辑更新

### 7.1 时间冲突检测

**新的冲突检测逻辑：**

```python
def check_time_conflict(student_id: str, start_time: datetime, end_time: datetime):
    """检查时间冲突"""
    appointments = get_appointments_by_date(start_time.date())

    for apt in appointments:
        apt_start = apt.get('start_time')
        apt_end = apt.get('end_time')

        # 检查时间重叠
        if (start_time < apt_end and end_time > apt_start):
            if apt.get('student_id') == student_id:
                raise ValueError(f"您在该时间段已有预约：{apt_start}-{apt_end}")

            # 检查1v1课程冲突
            student = get_student(student_id)
            conflict_student = get_student(apt.get('student_id'))
            if student['package_type'] == '1v1' and conflict_student['package_type'] == '1v1':
                raise ValueError(f"该时间段已被其他1v1预约占用：{apt_start}-{apt_end}")

    return True
```

### 7.2 动态状态计算

**更新动态状态逻辑：**
```python
def calculate_dynamic_status(start_time: datetime, end_time: datetime) -> str:
    """根据当前时间和课程时间计算动态状态"""
    now = datetime.now()

    if now < start_time:
        return "scheduled"    # 未开始
    elif now < end_time:
        return "active"        # 进行中
    else:
        return "completed"     # 已结束
```
