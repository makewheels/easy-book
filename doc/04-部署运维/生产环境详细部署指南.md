## 开发和修改流程

### 重要原则：本地优先原则
**所有代码和数据修改都必须遵循以下流程**：

1. **本地修改** - 在本地开发环境中进行所有修改
2. **本地验证** - 在本地环境充分测试，确保功能正常
3. **本地测试** - 运行完整的功能测试，验证修改没有破坏现有功能
4. **打包部署** - 将验证通过的代码打包部署到服务器
5. **服务器验证** - 在服务器环境进行最终验证

### 禁止操作
**严禁直接在服务器上进行以下操作**：
- ❌ 直接编辑服务器上的代码文件
- ❌ 直接在服务器数据库上执行修改命令
- ❌ 在服务器上调试或开发新功能
- ❌ 跳过本地验证直接修改服务器


#### 步骤3: Nginx配置
```bash
# 1. 创建nginx配置文件 (必须使用sudo)
sudo tee /etc/nginx/sites-enabled/easy-book.conf << 'EOF'
server {
    listen 80 default_server;
    server_name _;

    # API请求代理到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 30s;
        proxy_send_timeout 30s;
        proxy_read_timeout 30s;
    }

    # 健康检查
    location /health {
        proxy_pass http://127.0.0.1:8002/health;
        proxy_set_header Host $host;
    }

    # 前端静态文件 (重要: 使用/var/www/html路径)
    location / {
        root /var/www/html;
        try_files $uri $uri/ /index.html;
        index index.html;
    }

    # 静态资源缓存优化
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /var/www/html;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
EOF

# 2. 删除默认配置 (避免冲突)
sudo rm -f /etc/nginx/sites-enabled/default

# 3. 检查nginx配置
sudo nginx -t

# 4. 重新加载nginx配置
sudo systemctl reload nginx
```

### 关键注意事项

1. **权限问题**: 所有nginx相关操作必须使用`sudo`
2. **前端路径**: 必须使用`/var/www/html`，不要使用用户目录
3. **文件权限**: nginx静态文件必须设置为`www-data:www-data`或`755`权限
4. **环境变量**: 后端`.env`文件必须包含正确的MongoDB连接字符串

## 常见问题解决

### 1. 前端页面500错误 (权限问题)
**症状**: 访问主页显示 "500 Internal Server Error"
**原因**: nginx无法访问前端文件，通常是权限或路径问题
**解决方案**:
```bash
# 检查nginx错误日志
sudo tail -10 /var/log/nginx/error.log

# 修复文件权限 (必须使用sudo)
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html

# 检查nginx配置路径是否正确
sudo grep -n "root /var/www/html" /etc/nginx/sites-enabled/easy-book.conf

# 重新加载nginx
sudo systemctl reload nginx
```

### 2. 后端无法启动
- 检查端口是否被占用:
  - Windows: `netstat -ano | findstr :8002`
  - Linux: `netstat -tlnp | grep 8002`
- 如果端口被占用，需要杀掉进程:
  - Windows: `taskkill /F /PID <进程ID>`
  - Linux: `kill -9 <进程ID>`
- 查看错误日志: `tail -f /tmp/backend.log`
- 检查环境变量和配置文件

### 3. API 404错误
- 检查nginx配置是否正确加载
- 验证nginx主配置是否包含 `include /etc/nginx/sites-enabled/*;`
- 测试后端直接访问: `curl http://localhost:8002/api/students/`

### 4. 数据库连接问题
- 检查MongoDB服务状态
- 验证连接字符串格式
- 检查数据库索引和权限

### 5. 前端页面显示旧版本
- 清除浏览器缓存 (Ctrl+F5)
- 检查前端文件是否正确部署到 `/var/www/html`
- 验证nginx配置路径指向正确位置

### 6. 签到/缺席后按钮不消失
- 检查后端API返回的状态是否正确（should be 'checked' or 'absent'）
- 验证前端是否正确接收到更新后的状态数据
- 强制刷新浏览器页面（Ctrl+F5）清除缓存
- 检查前端状态更新逻辑：`student.status === 'scheduled'`

## 数据库索引管理

### 项目设计原则
本项目遵循**不在数据库层面使用唯一性约束**的设计原则：
- 仅创建查询性能优化索引
- 通过业务逻辑层处理数据唯一性验证
- 避免数据库约束带来的部署复杂性问题

### 索引系统
- **自动创建**: 应用启动时自动创建所需的查询优化索引
- **模型管理**: 索引定义在MongoDB模型类中 (backend/api_server/models.py)
- **禁止唯一约束**: 系统强制禁止创建唯一索引
