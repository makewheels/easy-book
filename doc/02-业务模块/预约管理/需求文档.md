# 表格日历学生展示功能需求文档

## 一、功能概述

表格日历功能是对现有预约管理系统的重要补充，以日历表格形式直观展示学生的预约安排，便于教练快速查看和管理整体排课情况。

## 二、核心需求

### 1. 日历表格展示
- **时间维度**：按周展示（7天 × 时间段）
- **时间段**：显示当天所有预约时段（如08:00、09:00、10:00等）
- **学生信息**：在每个时间格中显示预约的学生姓名
- **套餐标识**：用不同颜色或标签区分1v1和1v多套餐类型

### 2. 交互功能
- **日期导航**：支持切换到前一周/后一周
- **快速跳转**：支持跳转到"本周"、"下周"等快捷日期
- **学生详情**：点击学生姓名可跳转到该学生详情页
- **单元格操作**：点击空白时间格可快速创建预约

### 3. 数据展示
- **多学生显示**：同一时间段多个学生用列表形式展示
- **冲突检测**：1v1课程冲突时用特殊标识提醒
- **状态指示**：显示学生是否已签到、缺席等状态

## 三、界面布局

### 1. 顶部操作区
- 左侧：日期导航按钮（上一周/下一周）
- 中间：当前显示的日期范围（如"2024年12月2日 - 12月8日"）
- 右侧：快捷跳转按钮（本周、下周）

### 2. 表格结构
```
| 时间 | 周一(12/2) | 周二(12/3) | 周三(12/4) | 周四(12/5) | 周五(12/6) | 周六(12/7) | 周日(12/8) |
|------|------------|------------|------------|------------|------------|------------|------------|
| 08:00 | 张三(1v1) |            | 李四(1v多) |            | 王五(1v1) |            | 张三,李四 |
| 09:00 |            | 赵六(1v1) |            | 张三(1v1) |            | 李四(1v多) |            |
```

### 3. 底部导航
保持现有的底部导航栏，确保用户体验一致性

## 四、技术需求

### 1. 数据接口
- **预约数据**：按日期范围查询预约信息
- **学生信息**：获取学生基本信息和套餐类型
- **时间配置**：获取可选的时间段列表

### 2. 性能要求
- **加载速度**：页面加载时间不超过2秒
- **响应性**：日期切换流畅无明显延迟
- **数据缓存**：合理缓存已加载的预约数据

### 3. 移动端适配
- **横向滚动**：支持左右滑动查看不同日期
- **触摸优化**：单元格点击区域不小于44px
- **响应式布局**：适配不同屏幕尺寸

## 五、用户体验要求

### 1. 视觉设计
- **简洁清晰**：避免信息过载，重点突出
- **色彩搭配**：1v1和1v多套餐使用不同颜色区分
- **状态反馈**：操作后及时给出视觉反馈

### 2. 操作便利
- **快速导航**：支持快速跳转到指定日期
- **便捷操作**：支持长按、双击等快捷操作
- **错误处理**：网络异常时友好提示

### 3. 信息层次
- **主要信息**：学生姓名、套餐类型
- **次要信息**：学习项目、预约状态
- **详细信息**：点击后显示更多详情

## 六、业务逻辑

### 1. 预约规则
- 保持现有的1v1冲突检测机制
- 1v多课程允许同时间段多个学生
- 新建预约时检查时间冲突

### 2. 状态管理
- 实时显示预约的最新状态
- 支持从日历页面快速修改预约
- 签到状态及时更新到日历视图

### 3. 数据一致性
- 确保日历数据与学生详情页数据一致
- 预约修改后及时刷新日历显示
- 多端数据同步

## 七、非功能性需求

### 1. 可访问性
- 支持键盘导航
- 色彩对比度符合无障碍标准
- 重要信息不仅依赖颜色区分

### 2. 兼容性
- 支持主流移动浏览器
- 兼容不同屏幕分辨率
- 支持横竖屏切换

### 3. 安全性
- 数据传输加密
- 防止未授权访问
- 操作日志记录

## 十、状态颜色规范与显示规则

### 1. 状态类型与视觉规范
- **未到上课时间**：蓝色背景（#1890ff），黑色文字（#000），字体加粗
- **到了上课时间**：橙色背景（#fa8c16），黑色文字（#000），字体加粗
- **已签到**：绿色背景（#52c41a），黑色文字（#000），字体加粗
- **已缺席**：灰色背景（#8c8c8c），黑色文字（#000），字体加粗

### 2. 显示规则
- 仅显示学生姓名，不显示状态文字标签
- 通过背景色和文字颜色区分不同状态
- 所有时间段都必须有状态，不允许空白显示
- 状态颜色应与业务含义保持一致：蓝色表示待处理，橙色表示警告，绿色表示完成，灰色表示失效
- 使用高对比度设计：深色背景配黑色文字，确保文字清晰可读
