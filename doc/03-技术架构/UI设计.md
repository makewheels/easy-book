# 泳课学员管理系统 - UI设计文档

## 页面结构概览

本系统共包含4个页面，针对手机端设计：

1. **首页 - 预约页面**
2. **学生管理页面**
3. **学生详情/编辑页面**
4. **新增学生页面**

---

### 功能说明
- **日期显示**：按日期分组显示所有未来预约，格式为"MM-DD 周X"
- **时间显示**：时间在左侧，只显示白天工作时间（8:00-18:00）
- **预约展示**：每个时间段显示该时间的所有学生
- **签到功能**：
  - 未上的课：显示[签到]（绿色）[缺席]（红色）按钮
  - 已上的课：显示"已签到"状态或"已缺席"状态
  - 1v1学生：同一时间只有一个
  - 1v多学生：同一时间可以有多个
- **底部导航**：预约管理、学生管理
- **点击学生**：跳转到学生详情页面

### 功能说明
- **基本信息**：可编辑的姓名、别称、学习项目（带快捷选项）、备注

- **上课统计**：剩余次数、已上课次数、总次数
- **预约记录**：
  - 显示该学生的所有预约记录
  - 每条记录可以单独签到
  - 每条记录可以编辑时间
- **历史记录**：显示上课签到历史
- **保存按钮**：保存所有修改的信息

## 页面跳转关系

```
首页(预约管理) ←→ 学生管理
    ↓           ↓
学生详情 ←→ 学生详情
    ↑
新增学生
```

1. 首页 → 点击学生 → 学生详情
2. 首页 → 学生管理
3. 学生管理 → 点击学生 → 学生详情
4. 学生管理 → 新增学生
5. 学生详情 → 返回 → 上一页

---

## 设计要点

1. **手机端优化**：所有页面适合手机屏幕操作
2. **简洁明了**：信息层次清晰，操作简单
3. **快速签到**：首页和详情页都支持快速签到
4. **数据实时**：剩余次数等信息实时更新
5. **防误操作**：重要操作有确认提示

---

## 技术实现建议

1. **前端框架**：建议使用Vue.js或React
2. **UI组件库**：建议使用Vant或Ant Design Mobile
3. **数据存储**：本地存储 + 云端同步
4. **响应式设计**：确保不同手机屏幕适配
