## 🏗️ 技术架构

### 后端架构 (FastAPI)
- **框架**: FastAPI 0.104+
- **数据库**: MongoDB
- **ORM**: PyMongo
- **端口**: 8002

### 前端架构 (Vue 3)
- **框架**: Vue 3 + Composition API
- **构建工具**: Vite
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP客户端**: Axios
- **端口**: 5173

### 数据库设计 (MongoDB)
- **集合**: students, appointments, attendances
- **索引策略**: 非唯一索引，支持云端部署
- **数据验证**: Pydantic模型

## 🔄 系统工作流程

### 学员管理流程
1. 前端页面 → 调用学员API
2. FastAPI接收请求 → 数据验证
3. MongoDB存储/查询数据
4. 返回结果 → 前端更新UI

### 预约管理流程
1. 创建预约 → 冲突检测
2. 数据库存储 → 状态管理
3. 日历显示 → 实时更新

### 考勤管理流程
1. 签到/缺席操作
2. 课程数量自动扣减
3. 考勤记录存储
4. 统计数据更新

## 📱 移动端设计

- **响应式设计**: 最大宽度430px
- **触摸友好**: 最小44px点击区域
- **底部导航**: 便于单手操作
- **垂直布局**: 避免横向滚动

## 🧪 测试架构

- **测试框架**: pytest + Selenium
- **测试类型**: API测试 + UI测试 + 集成测试
- **测试覆盖**: 学员管理、预约管理、考勤管理
- **独立运行**: 可脱离主项目单独执行

## 🚀 部署架构

### 本地开发
- 后端: Python开发服务器 (8002端口)
- 前端: Vite开发服务器 (5173端口)
- 数据库: 本地MongoDB实例

### 生产环境
- 服务器: 云服务器 (Ubuntu)
- 前端: Nginx静态文件服务 (80端口)
- 后端: Gunicorn + Uvicorn (8002端口)
- 数据库: MongoDB云数据库
- 反向代理: Nginx
