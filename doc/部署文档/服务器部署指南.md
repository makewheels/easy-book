# Easy Book 服务器部署完整指南

## 推荐方案：Docker + CI/CD 流水线

这是最适合生产环境的方案，具有自动化、可回滚、易维护等优点。

### 方案一：GitHub Actions + Docker（推荐）

#### 1. 准备工作

```bash
# 1. 在服务器上安装 Docker 和 Docker Compose
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 2. 克隆代码到服务器
git clone https://github.com/your-username/easy-book.git
cd easy-book

# 3. 配置环境变量
cp .env.example .env
nano .env  # 编辑配置
```

#### 2. 环境变量配置

```env
# 生产环境数据库配置
MONGODB_URL=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/easy_book?retryWrites=true&w=majority
DB_NAME=easy_book
MONGO_ROOT_USERNAME=admin
MONGO_ROOT_PASSWORD=your_secure_password

# 或者使用本地 Docker MongoDB
# MONGODB_URL=mongodb://admin:password123@mongodb:27017/easy_book?authSource=admin
```

#### 3. GitHub Secrets 配置

在 GitHub 仓库设置中添加以下 Secrets：

- `HOST`: 你的服务器 IP 或域名
- `USERNAME`: 服务器登录用户名
- `SSH_KEY`: 服务器 SSH 私钥
- `DOMAIN`: 你的域名（可选）

#### 4. 首次部署

```bash
# 在服务器上运行
chmod +x deploy.sh
./deploy.sh
```

#### 5. 自动化部署

配置完成后，每次推送代码到 main 分支，GitHub Actions 会自动：
- 运行测试
- 构建镜像
- 部署到服务器
- 进行健康检查

### 方案二：手动 Docker 部署

如果不使用 CI/CD，可以手动部署：

```bash
# 1. 更新代码
git pull origin main

# 2. 重新构建并启动
docker-compose down
docker-compose up -d --build

# 3. 查看状态
docker-compose ps
docker-compose logs -f
```

### 方案三：传统部署（不使用 Docker）

#### 1. 安装环境

```bash
# 安装 Python 3.11
sudo apt update
sudo apt install python3.11 python3.11-venv python3.11-pip

# 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 Nginx
sudo apt install nginx

# 安装 MongoDB
wget -qO - https://www.mongodb.org/static/pgp/server-7.0.asc | sudo apt-key add -
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list
sudo apt-get update
sudo apt-get install -y mongodb-org
```

#### 2. 部署后端

```bash
# 创建应用目录
sudo mkdir -p /opt/easy-book
sudo chown $USER:$USER /opt/easy-book
cd /opt/easy-book

# 克隆代码
git clone https://github.com/your-username/easy-book.git .

# 后端设置
cd backend
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 创建环境变量
cp ../.env.example .env
nano .env  # 编辑配置

# 测试运行
python run.py

# 使用 PM2 管理进程
sudo npm install -g pm2
pm2 start run.py --name "easy-book-backend"
pm2 save
pm2 startup
```

#### 3. 部署前端

```bash
# 回到项目根目录
cd /opt/easy-book/frontend

# 安装依赖
npm install

# 修改 API 地址（如果需要）
# 编辑 vite.config.js

# 构建生产版本
npm run build

# 复制到 Nginx 目录
sudo cp -r dist/* /var/www/html/
```

#### 4. 配置 Nginx

```bash
# 创建 Nginx 配置
sudo nano /etc/nginx/sites-available/easy-book
```

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:8002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```bash
# 启用站点
sudo ln -s /etc/nginx/sites-available/easy-book /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 数据库选择

### 选项 1：MongoDB Atlas（推荐）
- 免费套餐足够使用
- 自动备份
- 高可用性
- 无需维护

```env
MONGODB_URL=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/easy_book?retryWrites=true&w=majority
```

### 选项 2：自建 MongoDB
- 完全控制
- 可能更便宜（大流量时）
- 需要自己维护和备份

### 选项 3：Docker MongoDB
- 适合开发和小规模部署
- 数据持久化需要配置卷

## SSL 证书配置

### 使用 Let's Encrypt（免费）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo crontab -e
# 添加：0 12 * * * /usr/bin/certbot renew --quiet
```

## 监控和维护

### 1. 日志监控

```bash
# Docker 日志
docker-compose logs -f backend
docker-compose logs -f frontend

# PM2 日志（传统部署）
pm2 logs easy-book-backend

# Nginx 日志
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

### 2. 性能监控

```bash
# 系统资源
htop
df -h
free -h

# Docker 资源
docker stats

# PM2 监控
pm2 monit
```

### 3. 备份策略

```bash
# MongoDB 备份脚本
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p /backup/mongodb
mongodump --uri "your_mongodb_uri" --out /backup/mongodb_$DATE

# 自动备份（每天凌晨2点）
sudo crontab -e
# 添加：0 2 * * * /path/to/backup_script.sh
```

## 故障排查

### 常见问题

1. **端口被占用**
   ```bash
   sudo netstat -tlnp | grep :8002
   sudo kill -9 PID
   ```

2. **权限问题**
   ```bash
   sudo chown -R $USER:$USER /opt/easy-book
   ```

3. **内存不足**
   ```bash
   # 增加 swap
   sudo fallocate -l 2G /swapfile
   sudo chmod 600 /swapfile
   sudo mkswap /swapfile
   sudo swapon /swapfile
   ```

4. **数据库连接失败**
   ```bash
   # 检查网络连接
   telnet mongodb-host 27017
   
   # 检查认证
   mongo "mongodb://username:password@host:port/database"
   ```

## 安全建议

1. **防火墙配置**
   ```bash
   sudo ufw enable
   sudo ufw allow ssh
   sudo ufw allow 80
   sudo ufw allow 443
   ```

2. **定期更新**
   ```bash
   sudo apt update && sudo apt upgrade
   docker-compose pull
   docker-compose up -d
   ```

3. **访问控制**
   - 使用强密码
   - 限制 SSH 访问
   - 使用 fail2ban
   - 定期检查日志

## 推荐的部署流程

1. **开发环境**：使用 Docker Compose 本地开发
2. **测试环境**：使用 GitHub Actions 自动测试
3. **生产环境**：使用 GitHub Actions + Docker 自动部署
4. **监控**：配置日志收集和告警
5. **备份**：设置自动备份策略

这样可以确保开发、测试、生产环境的一致性，同时实现自动化部署和维护。
