# 自动化测试说明

## 概述
本目录包含针对泳课预约系统学生管理模块的自动化测试脚本，主要用于测试身份证号码和手机号功能。

## 测试文件

### 1. `simple_test.py` (推荐使用)
简单易用的测试脚本，测试学生管理的核心功能：

- **连接测试**: 检查后端服务是否正常运行
- **创建学生**: 测试创建学生并验证身份证和手机号字段
- **获取学生**: 测试获取学生信息并验证身份证和手机号
- **更新学生**: 测试更新学生的身份证和手机号
- **删除学生**: 清理测试数据

### 2. `test_student_management.py`
功能完整的测试脚本，提供详细的测试报告和错误诊断。

### 3. `run_tests.py`
测试运行器，自动检查依赖并运行测试。

## 使用方法

### 前提条件
1. 确保后端服务正在运行（默认端口8003）
2. 安装Python依赖：
   ```bash
   pip install requests
   ```

### 运行简单测试
```bash
python simple_test.py
```

### 运行完整测试
```bash
python run_tests.py
```

或直接运行：
```bash
python test_student_management.py
```

### ✅ 身份证号码功能
- 创建学生时正确保存身份证号码
- 获取学生时正确返回身份证号码
- 更新学生时正确修改身份证号码

### ✅ 手机号码功能
- 创建学生时正确保存手机号码
- 获取学生时正确返回手机号码
- 更新学生时正确修改手机号码

## 故障排查

### 常见问题

1. **连接失败**
   - 检查后端服务是否启动
   - 确认端口配置（默认8003）

2. **创建学生失败**
   - 检查学生姓名是否唯一（数据库有唯一索引约束）
   - 确认必填字段是否完整

3. **更新/删除失败**
   - 确认学生ID是否正确
   - 检查API权限和数据库连接

### 调试方法
1. 查看后端日志：`tail -f backend/logs/app.log`
2. 检查数据库连接：`curl http://localhost:8003/health`
3. 手动测试API：使用Postman或curl

## 自动化建议

建议将此测试集成到CI/CD流程中：

1. **开发阶段**: 每次提交代码前运行测试
2. **部署前**: 部署到测试环境后运行测试
3. **定期测试**: 设置定时任务每天运行测试

## 测试数据说明

测试脚本会创建临时测试数据，并在测试结束时自动清理。如果测试中断，可能需要手动清理：

```bash
# 清理名为"TestStudent_"开头的测试学生
curl -X GET "http://localhost:8003/api/students/" | grep "TestStudent_"
```
